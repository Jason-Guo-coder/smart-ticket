# SmartTicket 智工单 · 设计规范（DESIGN.md）

> 版本：v1.0｜更新日期：2026-06-21
>
> 本文件是前端视觉与交互的唯一事实来源，基于已完成的 Figma 原型（13 个页面，Web 桌面端）。
> Figma 文件：`TnvlYjEKdMIRNeeWSM8NFM` · https://www.figma.com/design/TnvlYjEKdMIRNeeWSM8NFM
>
> 设计语言一句话：**iOS 风格的玻璃拟态（Glassmorphism）+ Apple 系统级配色 + 彩色发光背景 + 大圆角**。新增 / 修改任何界面都必须沿用此语言，不得引入新的设计风格或后台管理模板风。

---

## 1. 设计原则

1. **一致性优先**：所有页面复用同一套 token、组件与布局骨架，不为单页造新样式。
2. **玻璃拟态克制使用**：半透明白 + 背景模糊用于卡片 / 侧边栏 / 顶栏；正文与数据保持高对比、清晰可读。
3. **苹果系统观感**：大圆角、轻投影、留白充足、动效柔和。
4. **后端为主、前端能演示**：组件以 Element Plus 为底，按本规范覆盖样式即可，不在样式上过度纠结。

## 2. 主色调与设计令牌（Design Tokens）

> Figma 文件内已建变量集合 `SmartTicket Tokens`。前端落地时建议映射为 CSS 变量 / Tailwind theme。

### 2.1 品牌与中性色

| Token | 值 | 用途 |
|---|---|---|
| `apple/blue` | `#0071e3` | 主色：主按钮、链接、激活态、强调 |
| `apple/text` | `#1d1d1f` | 主文字 |
| `apple/darkgray` | `#86868b` | 次要文字、占位符、说明 |
| `apple/gray` | `#f5f5f7` | 浅色分隔 / 次级背景 |
| `apple/white` | `#ffffff` | 卡片基底（配透明度使用） |
| 页面底色 | `#fbfbfd` | 屏幕画布背景 |

### 2.2 工单状态色（语义色）

| 状态 | Token | 值 |
|---|---|---|
| 待派单 | `status/pending` | `#f5a623`（琥珀） |
| 处理中 | `status/processing` | `#0071e3`（蓝） |
| 待验收 | `status/accepting` | `#af52de`（紫） |
| 已完成 | `status/done` | `#34c759`（绿） |
| 已评价 | `status/rated` | `#5ac8fa`（青蓝） |
| 已驳回 / 异常 | `status/rejected` | `#ff3b30`（红） |

### 2.3 优先级色

| 优先级 | Token | 值 |
|---|---|---|
| 高 | `priority/high` | `#ff3b30` |
| 中 | `priority/mid` | `#f5a623` |
| 低 | `priority/low` | `#34c759` |

### 2.4 字体与字号

- **字体族**：界面以中文为主，统一使用 **Noto Sans SC**（落地可回退到 `-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`）。
- **字重**：Regular（正文）/ Medium（强调、按钮、激活态）/ Bold（标题）。
- **字号阶梯**：

| 用途 | 字号 | 字重 |
|---|---|---|
| 页面主标题 | 22 | Bold |
| 卡片标题 | 16–18 | Bold |
| 正文 | 13–14 | Regular |
| 次要 / 说明 / 表头 | 11–12 | Regular |
| KPI 数字 | 26–40 | Bold |
| 按钮 | 14–16 | Medium |

### 2.5 圆角、间距、阴影、模糊

| 项 | 规范 |
|---|---|
| 圆角 | 卡片 20–24px；按钮 / 输入框 12–16px；徽章 8px；胶囊 / 头像 全圆 |
| 间距 | 内容区 padding 32；卡片 padding 22；卡片间距 16–20；元素间距 8–14 |
| 投影 | 卡片：`0 12 32 rgba(0,0,0,0.08)`；按钮主色：`0 8 20 rgba(0,113,227,0.35)` |
| 背景模糊 | 玻璃卡 `blur(30)`；侧边栏 `blur(30)`；弹窗 `blur(40)` |

> ⚠️ **玻璃卡铁律**：圆角 + 投影的卡片，`clipsContent` 必须为 false，否则投影会被裁成方角矩形（已在原型踩坑）。

### 2.6 玻璃拟态层级

| 层级 | 背景 | 用途 |
|---|---|---|
| `glass-card` | 白 0.72 + blur(30) + 白描边 0.7 | 主内容卡片 |
| 侧边栏 | 白 0.55 + blur(30) | 左侧导航 |
| 顶栏控件 | 白 0.6 + blur | 搜索框 / 铃铛 |
| AI 面板 | 蓝紫渐变 0.85 + blur | AI 智能分析专属 |
| 弹窗 | 白 0.85 + blur(40) + 强投影 | 模态框 |

### 2.7 彩色发光背景（Blob）
每个屏幕固定 4 个高斯模糊（`blur(90)`）的彩色圆形光晕，透明度 0.3–0.5：左上**蓝**、右上**紫**、左下**粉**、右下**橙**。营造苹果发布会式的氛围背景，所有玻璃卡片浮于其上。

## 3. 布局骨架

```
┌──────────┬─────────────────────────────────────────┐
│          │  顶栏：页面标题 + 副标题 ┄┄┄ 搜索 铃铛 头像   │ 76px
│  侧边栏   ├─────────────────────────────────────────┤
│  248px   │                                          │
│  玻璃     │   内容区（padding 32，玻璃卡片纵向排布）      │
│  导航     │                                          │
│  +用户卡  │                                          │
└──────────┴─────────────────────────────────────────┘
        （整屏浮于彩色 blob 背景之上）
```

- **桌面端基准宽度 1440px**；内容区自适应填充。
- **登录页例外**：无侧边栏，居中玻璃卡片。
- **侧边栏导航按角色不同**：报修人（提交工单 / 我的工单）；工程师（待办工单 / 工单处理 / 我的绩效）；管理员（工单看板 / 派单管理 / 工程师管理 / 用户与角色 / 审计日志）。

## 5. 关键交互动作

| 交互 | 触发 | 表现 |
|---|---|---|
| **卡片悬浮 hover-lift** | 鼠标悬停可点击卡片 | 上移 6px + 投影加深 + 背景提亮，0.3s ease |
| **侧边栏切换** | 点击导航项 | 目标项变激活态（蓝底蓝字），内容区 0.4s 淡入上移 |
| **下拉菜单** | 悬停 / 点击 | 玻璃下拉，0.2s 淡入下移 |
| **采纳 AI 建议** | 派单页点击「采纳 AI 建议派单」 | 按 AI 建议工程师即时派单，工单移出待派单列表，状态→处理中 |
| **一键智能分派** | 派单页右上按钮 | 批量按 AI 建议 + 负载分派全部待派单 |
| **状态流转** | 工程师/管理员操作 | 按状态机执行，时间线追加一条节点；非法流转拦截并提示 |
| **星级评分** | 评价弹窗点击星 | 实时高亮 1–5 星 + 文案（如"非常满意"） |
| **密码可见切换** | 登录页眼睛图标 | 明文 / 密文切换 |
| **SLA 倒计时** | 工单详情 / 看板 | 实时剩余时间；临近 / 超时变橙 / 红 |

## 6. 状态视觉表现（错误 / 加载中 / 空数据）

> 三态必须在所有列表、表单、AI 面板、看板中实现，保持视觉统一。

### 6.1 加载中（Loading）
- **列表 / 表格**：玻璃卡内骨架屏（灰色圆角占位条，轻微呼吸动效）。
- **AI 分析面板**：火花图标旋转 + "AI 正在分析…"，结果行用骨架占位；标题右侧状态徽章显示"分析中"（蓝）。
- **按钮**：点击后转 loading 态（spinner + 禁用），文案不变或加"…"。
- **看板 / 图表**：卡片整体骨架占位，不显示 0 误导。

### 6.2 空数据（Empty）
- 居中插画 / 大图标（灰）+ 一句说明 + 可选主操作按钮。
- 文案示例：
  - 我的工单：「还没有工单，去提交一个吧」+「提交工单」按钮。
  - 待办工单：「太棒了，当前没有待办」。
  - 派单页：「暂无待派单工单」。
  - 审计日志 / 搜索：「没有匹配的记录，换个条件试试」。

### 6.3 错误（Error）
- **接口错误**：玻璃卡内红色图标 + 错误说明 + 「重试」按钮（红描边次按钮）。
- **AI 失败降级**：面板显示中性提示「AI 暂时不可用，已转人工派单」，状态徽章变灰"已降级"，**不报红、不阻断**主流程（呼应 PRD 的降级原则）。
- **表单校验**：输入框红描边 + 下方红色错误文字（13px）。
- **权限不足（403）**：整页/区块提示「无访问权限」。
- **全局提示**：操作成功 / 失败用 Element Plus message（成功绿、失败红、警告橙），右上角短暂出现。

### 6.4 SLA / 超时态
- 临近超时：橙色徽章 + 橙色进度；已超时：红色徽章「超时 Xh Ym」+ 看板该行高亮。

## 7. 可访问性与细节底线
- 正文与背景对比度满足可读性（玻璃卡上文字保持深色）。
- 可点击元素有明确 hover / active 反馈。
- 状态不只靠颜色区分：状态色同时配文字标签（色盲友好）。
- 图标统一线性风格（line icon），线宽 2，关键尺寸 16 / 18 / 20 / 24。
